## Setup

### 1. Database

Run all three SQL files in DBeaver in order:
1. `migrate.sql`
2. `migrate_models.sql`
3. `migrate_prompts.sql`

Then run these two ALTER statements to add rating support:
```sql
ALTER TABLE loras  ADD COLUMN rating TINYINT DEFAULT 0;
ALTER TABLE models ADD COLUMN rating TINYINT DEFAULT 0;
```

### 2. Environment

Copy `backend/.env.example` to `backend/.env` and fill in your values:

```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=comfyui_assets
LORA_FOLDER=C:\path\to\ComfyUI\models\loras
CHECKPOINT_FOLDER=C:\path\to\ComfyUI\models\checkpoints
VAE_FOLDER=C:\path\to\ComfyUI\models\vae
UPSCALER_FOLDER=C:\path\to\ComfyUI\models\upscale_models
DIFFUSION_FOLDER=C:\path\to\ComfyUI\models\diffusion_models
```

### 3. Dependencies

```bash
pip install flask flask-cors mysql-connector-python python-dotenv pillow
cd frontend && npm install
```

### 4. Launch

Double-click `start.bat` вЂ” scans all models, starts API + UI.

## Prompt Gallery

Drop any PNG generated by ComfyUI into the Prompt Gallery tab. The app reads the embedded workflow JSON and automatically extracts:

- Positive and negative prompts
- Checkpoint / base model name
- All LoRAs used with their strengths
- Seed, sampler, scheduler, steps, CFG, resolution

You can edit any field before saving, and add personal notes to each entry. Entries are filterable by base model and searchable by prompt text, LoRA name, or notes.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/loras` | All LoRAs |
| GET | `/models?type=checkpoint` | Models filtered by type |
| GET | `/tags` | All tags |
| POST | `/resync` | Rescan LoRA folder |
| POST | `/resync_models` | Rescan all model folders |
| POST | `/set_rating` | Set star rating for lora or model |
| GET | `/prompts` | All prompt gallery entries |
| POST | `/prompts/extract` | Extract metadata from PNG (file upload or path) |
| POST | `/prompts/save` | Save prompt gallery entry |
| POST | `/prompts/delete` | Delete prompt entry |
| POST | `/drop_image` | Move image to model folder + register in DB |
| GET | `/preview?path=...` | Serve any local image file |
| POST | `/open_folder` | Open model folder in Explorer |

## Notes

- `.env` is gitignored вЂ” your password never leaves your machine
- Base model is auto-detected from folder names, overridable manually
- Any custom base model you type is saved and reusable across models
- Prompt Gallery reads ComfyUI's native PNG workflow chunks вЂ” no A1111 support needed
- `pillow` is required for future image processing; install it alongside the other dependencies
